# Values

利便性と安全性が競合したときは、次の基準で判断する。

## Skillは使用中に育つ

実際のprojectで使い、足りない点をその場で直せることを優先する。

## 変更を先に見えるようにする

同期の前に、localとsourceのどちらが変わったかを確認できるようにする。

## 曖昧な判断は人に残す

競合、上書き、履歴変更を推測で進めない。判断に必要な差分を見える形で残す。

## 人とagentが同じ流れを使う

専用概念を増やさず、短い共通workflowで運用する。
